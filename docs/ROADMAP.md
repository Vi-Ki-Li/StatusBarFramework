# 开发路线图

> 详细阶段规划见 [docs/phases/](./phases/) 目录。当前进度见 [STATE.md](./STATE.md)。

## 阶段总览

```
Phase 0: 基础设施        ──── 脚本骨架、生命周期、安全模式、CSS体系
Phase 1: 数据层核心      ──── 数据模型、JSON Patch解析、角色ID系统
Phase 2: 数据工作室      ──── 条目字典、分类管理、元数据编辑
Phase 3: 状态栏渲染引擎  ──── 模板引擎、默认样式、多角色切换
Phase 4: 样式工坊        ──── 样式编辑器、实时预览、CSS安全检查
Phase 5: 布局编排器      ──── WYSIWYG画布、拖拽排列、JSON序列化
Phase 6: 数据中心        ──── 全功能编辑器、暂存/保存、智能推荐
Phase 7: 管理器整合      ──── 浮窗壳、模块路由、深度链接
Phase 8: 系统配置        ──── 主题组合、叙事快照、备份迁移
Phase 9: 高级功能        ──── 游戏交互、关系图谱、AI辅助
Phase 10: 打磨优化       ──── 响应式、性能、用户文档
```

## 依赖关系

```
P0 ─→ P1 ─→ P2 ─→ P3 ─┐
                        ├─→ P7 ─→ P8 ─→ P9 ─→ P10
               P4 ──────┤
               P5 ──────┘
               P6 ──────┘
```

- Phase 0-1: 串行（基础→数据层）
- Phase 2: 依赖 Phase 1
- Phase 3: 依赖 Phase 2（需要条目定义来渲染）
- Phase 4-5-6: 可与 Phase 3 并行开发，互相独立
- Phase 7: 依赖 Phase 3-6 全部完成（整合所有模块）
- Phase 8-10: 串行

---

## Phase 0: 基础设施

**目标**: 搭建脚本骨架，建立所有后续开发的基础。

- [ ] 脚本入口 (`src/通用状态栏框架脚本/index.ts`)
- [ ] 单例所有权机制 (唯一ID + top窗口全局信标 + cleanup)
- [ ] 安全模式 (`Ctrl+Alt+Shift+R`)
- [ ] CSS 设计令牌体系 (CSS变量、深浅色主题、`omg-` 前缀)
- [ ] 基础组件库 (按钮、输入框、下拉框、开关、标签、图标)
- [ ] 管理器浮窗壳 (iframe 隔离 + extensionsMenu 按钮)
- [ ] Google Fonts 加载 (Inter)
- [ ] Font Awesome / Lucide 图标加载
- [ ] 工具函数 (CSS 注入、DOM 工具等)

## Phase 1: 数据层核心

**目标**: 建立数据模型和解析引擎。

- [ ] 核心 Zod Schema 定义 (角色数据、共享数据、元数据)
- [ ] 角色 ID 系统 (`char_id` 生成 + 映射表)
- [ ] JSON Patch (RFC 6902) 解析器
- [ ] 数据读写封装 (脚本变量 / 聊天变量 / 消息楼层变量)
- [ ] 智能合并逻辑 (source_id + user_modified 标记)
- [ ] 会话完整性检测 (_meta.message_count)
- [ ] `nil` 值处理
- [ ] MVU 兼容适配层

## Phase 2: 数据工作室

**目标**: 数据的显示/验证元数据管理（不定义数据结构，JSON Patch 自动处理）。

- [ ] 分类管理 UI (CRUD + 共享/角色属性)
- [ ] 条目 CRUD UI (key=JSON路径 / name / category / icon)
- [ ] 图标选择器 (搜索 + 预览)
- [ ] data_type + 验证规则 → Zod Schema 自动生成
- [ ] ui_type 关联样式工坊
- [ ] interaction_type 关联交互行为
- [ ] description (AI指导文本) + update_sample (JSON Patch 示例自动生成)
- [ ] 世界书注入功能
- [ ] Other 回退显示 (未定义 key 自动归入)
- [ ] 导入/导出

## Phase 3: 状态栏渲染引擎

**目标**: 在聊天消息中渲染状态栏。

- [ ] {{placeholder}} 模板引擎
- [ ] 样式单元渲染器
- [ ] 布局 JSON → DOM 渲染
- [ ] 多角色切换 (横向滚动选择栏)
- [ ] 折叠/展开 (区块级 + 状态记忆)
- [ ] 事件绑定系统 (interaction_type → 动态绑定)
- [ ] 内置默认样式单元 (3-5套)
- [ ] 内置默认布局方案

## Phase 4: 样式工坊

**目标**: 创建和管理可复用样式。

- [ ] 样式单元 CRUD UI
- [ ] 手写代码编辑器 (HTML + CSS, 语法高亮?)
- [ ] {{placeholder}} 参考面板
- [ ] 实时预览区 (可交互 → 点击元素切换CSS)
- [ ] 可视化 GUI 配置面板
- [ ] 动态 CSS 文档 (解析 `/* @doc: */` 注释)
- [ ] 内置样式库 (只读 + 复制)
- [ ] CSS 安全检查
- [ ] 全局主题编辑器
- [ ] 拖拽应用样式
- [ ] 导入/导出

## Phase 5: 布局编排器

**目标**: 可视化设计状态栏布局，支持多种 CSS 布局模式。

- [ ] 分层布局系统设计 (布局模式/容器嵌套/条目放置)
- [ ] 布局模式支持 (Flexbox / CSS Grid / Masonry / Absolute / Custom CSS)
- [ ] WYSIWYG 画布区域
- [ ] 组件列表侧边栏 (数据工作室条目)
- [ ] 拖拽条目到容器
- [ ] 容器嵌套 (树状结构, 每个容器独立布局模式)
- [ ] 元素检视面板 (尺寸/间距/对齐/跨度)
- [ ] 调整手柄
- [ ] 布局 JSON 序列化/反序列化 (树状结构)
- [ ] JSON 直接编辑模式 (高级用户)
- [ ] Custom CSS 模式
- [ ] 导入/导出

## Phase 6: 数据中心

**目标**: 全功能实时数据编辑器。

- [ ] 分层数据展示 UI (世界状态/角色状态分区)
- [ ] 角色数据编辑 (char_id 显示)
- [ ] 标签双击编辑 / "x"删除 / 行级删除 / 区级新增
- [ ] 智能推荐输入框
- [ ] 暂存模式 + 全局保存
- [ ] isPresent 切换 (眼睛图标)
- [ ] Other 回退分类
- [ ] 清空数据 / 删除角色

## Phase 7: 管理器整合

**目标**: 将五大模块整合到管理器中。

- [ ] 管理器主壳 (Vue Router 选项卡)
- [ ] 五大模块路由注册
- [ ] 模块间深度链接 (定义→样式, 预设→样式)
- [ ] 帮助指引系统 (? 图标 → 浮窗)
- [ ] #extensionsMenu 按钮 → 打开管理器

## Phase 8: 系统配置

**目标**: 整合应用逻辑与系统级设置。

- [ ] 原始数据查看器 (DevTools 风格)
- [ ] 主题组合 CRUD (世界书+定义+样式+布局)
- [ ] 主题绑定/应用逻辑
- [ ] 叙事快照系统 (模板CRUD + 数据变化检测)
- [ ] 世界书条目管理
- [ ] 备份与迁移 (多选导出/导入/恢复出厂)
- [ ] 使用指南 (内置帮助)

## Phase 9: 高级功能

**目标**: 锦上添花的高级体验。

- [ ] 游戏式交互系统 (用户自定义事件脚本)
- [ ] 角色关系图谱 (星型/网状/组织架构)
- [ ] 叙事风格指导 (提示词注入)
- [ ] 一键导出为正则+世界书
- [ ] AI 辅助编写入口 (条目/样式/布局/完整配置)

## Phase 10: 打磨优化

**目标**: 产品化打磨。

- [ ] 响应式全面适配
- [ ] 性能优化 (懒加载、虚拟滚动等)
- [ ] 完整用户文档
- [ ] 导入/导出统一入口
- [ ] 边界情况处理
