# 需求分析文档

> 基于 [REQUIREMENTS-RAW.md](./REQUIREMENTS-RAW.md) 的结构化分析。设计决策见 [DECISIONS.md](./DECISIONS.md)。

## 1. 项目定位

**通用状态栏框架** — 一个运行在 SillyTavern 酒馆助手上的脚本，为用户提供高度可自定制的多角色状态数据展示与管理系统。

### 核心价值主张
- **零门槛**：不会前端的用户也能为任意角色卡创建状态栏
- **即插即用**：脚本自动管理所需的世界书、正则等依赖
- **绝对自由**：数据解析、UI渲染、布局、交互行为均可用户自定义
- **变量系统无关**：独立实现数据解析，兼容 MVU 及其他 JSON 变量系统

## 2. 产品组成

| 可视化布局   | P0     | 由"布局编排器"生成的 JSON 控制排列              |
| 多角色切换   | P0     | 横向滚动角色选择栏，显示 `isPresent: true` 角色 |

## 3. 后台核心系统

### 3.1 角色 ID 系统
- `char_id` 格式: `char_{标识}` (用户固定为 `char_user`)
- 映射表存储在聊天变量中: `{ char_id → 最新角色名 }`
- UI 动态读取角色名，`char_user` 通过酒馆宏获取

### 3.2 数据解析引擎
- **独立实现**，不强依赖 MVU
- 支持 JSON Patch (RFC 6902) 解析
- 提供 MVU 兼容接口（可接入 MVU 的 parseMessage/事件系统）
- 支持任何 JSON 格式的变量系统

### 3.3 智能合并（待简化）
- `source_id` 裁决: `Source.source_id >= Target.source_id` 才允许更新
- `user_modified` 标记: **不阻止 AI 更新**，改为通过叙事快照注入世界书告知 AI
- 优先级降级: 用户新消息时重置 `user_modified: false`
- `nil` 值: 渲染时隐藏，管理器可见

### 3.4 会话完整性
- `_meta.message_count` 记录消息数
- 检测"时间线收缩"并 `toastr.warning`

### 3.5 叙事生成器
- 检测数据变化 → 根据用户选择的模板生成叙事文本
- 模板使用 `{{placeholder}}` 语法
- 支持按数据类型/变化类型定制不同模板
- 可注入世界书发送给 AI

### 3.6 脚本生命周期
- 单例所有权: 唯一 ID + `top` 窗口全局信标
- 热重载: 新实例 cleanup 旧实例 → 声明所有权 → 初始化

## 4. 存储策略

| 数据类型              | 存储位置     | 说明                                       |
| --------------------- | ------------ | ------------------------------------------ |
| 条目定义 (数据工作室) | IndexedDB    | 跨聊天持久化，体积可能较大                 |
| 布局方案              | IndexedDB    | 布局 JSON 树可能较大                       |
| 样式单元 + 全局主题   | IndexedDB    | 包含 HTML/CSS 模板，体积可能较大           |
| 主题组合/预设         | IndexedDB    | 组合引用定义+样式+布局                     |
| 叙事模板              | IndexedDB    | 跨聊天持久化                               |
| 角色数据              | 聊天变量     | 绑定到当前聊天                             |
| 角色ID映射            | 聊天变量     | 绑定到当前聊天                             |
| 楼层快照              | 消息楼层变量 | 绑定到具体消息                             |
| 系统配置 (少量)       | 脚本变量     | 少量简单设置 (启用/禁用开关等)             |

## 5. 技术约束

| 约束     | 要求                                                 |
| -------- | ---------------------------------------------------- |
| CSS      | Vanilla CSS, BEM命名, `omg-` 前缀, 不使用酒馆CSS变量 |
| 字体     | Google Fonts (Inter/Roboto/Outfit)                   |
| 图标     | Font Awesome / Lucide, 不用 emoji                    |
| 语言     | UI 全部中文                                          |
| 响应式   | PC + 移动端                                          |
| 主题     | 深色/浅色切换                                        |
| 性能     | 语义化 HTML, 唯一 ID, 优化加载                       |
| iframe   | 不用 `vh` 单位, 用 `width + aspect-ratio`            |
| 安全模式 | `Ctrl+Alt+Shift+R` 强制恢复默认                      |

---

## 6. 跨模块需求

- **深度链接**: 所有模块间可快速跳转（如定义→样式编辑）
- **帮助指引**: 每个模块有 `?` 图标弹窗说明
- **导入/导出**: 所有模块支持独立导入/导出（全部/部分/单个）
- **深浅色**: 所有内置 UI 组件支持深浅色切换
- **用户可自定义**: 状态栏和管理器的 UI 都可由用户自定义外观
