# 测试指南

> 每个 Phase 完成后的测试方法。你需要在 SillyTavern 中加载脚本才能测试。

## 环境准备

1. 在项目根目录运行 `pnpm watch` 启动 webpack 热重载
2. 在 SillyTavern 中导入实时脚本（地址类似 `import 'http://localhost:5500/copilot-worktree-xxx/dist/通用状态栏框架脚本/index.js'`）
3. 打开浏览器开发者工具 (F12) 查看控制台输出

## Phase 0: 基础设施

### 测试清单

- [ ] 控制台显示 `[通用状态栏框架] 初始化，实例: xxx`
- [ ] 控制台显示 `extensionsMenu 按钮已添加`
- [ ] 打开 SillyTavern 右侧扩展菜单，能看到"状态栏管理器"按钮
- [ ] 点击"状态栏管理器"按钮，弹出管理器浮窗
- [ ] 管理器左侧有 5 个导航标签（数据中心、数据工作室、样式工坊、布局编排器、系统配置）
- [ ] 点击右上角深浅色切换按钮，主题切换正常
- [ ] 点击管理器外部区域（遮罩），管理器关闭
- [ ] 按 `Ctrl+Alt+Shift+R` 触发安全模式，控制台有日志
- [ ] 重新加载脚本时，旧实例被清理、新实例正常接管

## Phase 3: 状态栏渲染引擎

### 测试方法

状态栏需要聊天变量中有数据才会显示。测试步骤：

1. **先在数据工作室创建定义**
   - 新建分类"角色属性"（属性=角色）
   - 添加条目：key=`生命值`, name=`生命值`, dataType=`number`, validation.max=`100`
   - 添加条目：key=`好感度`, name=`好感度`, dataType=`number`, validation.max=`100`
   - 新建分类"世界状态"（属性=共享）
   - 添加条目：key=`时间`, name=`当前时间`, dataType=`text`

2. **写入测试数据到聊天变量**（在浏览器控制台执行）

```js
// 需要在酒馆主页面的控制台执行，或在脚本 iframe 中执行
insertOrAssignVariables({
  _omg_state: {
    _meta: { message_count: 1, version: 1 },
    _characters: {
      char_user: { char_id: 'char_user', name: '玩家', isPresent: true },
      char_npc1: { char_id: 'char_npc1', name: '少女A', isPresent: true }
    },
    shared: { 时间: '傍晚' },
    characters: {
      char_user: { 生命值: 85, 好感度: 60 },
      char_npc1: { 生命值: 70, 好感度: 45 }
    },
    _entry_meta: {}
  }
}, { type: 'chat' });
```

3. **发送一条消息**（让 AI 回复），状态栏应在最后一条消息下方显示 // 这个暂时不进行

### 测试清单

- [ ] 最后一条消息下方出现暗色玻璃拟态风格的状态栏
- [ ] 状态栏顶部显示角色切换标签栏（"玩家"和"少女A"）
- [ ] 默认显示第一个非用户角色的数据（少女A）
- [ ] 点击"玩家"标签，切换到玩家数据
- [ ] 分区（"角色属性"、"世界状态"）正确显示
- [ ] 数字类型条目显示进度条
- [ ] 文本类型条目显示为简单文本
- [ ] 点击分区标题，该分区折叠/展开
- [ ] 发送新消息后，状态栏自动刷新
- [ ] 未定义的数据条目归入"其他"分区

## 通用调试技巧

- **找不到 extensionsMenu 按钮**：检查 `#extensionsMenu` 元素是否存在，可能在不同酒馆版本中 ID 不同
- **管理器打不开**：查看控制台是否有 JS 错误
- **状态栏不显示**：确认聊天变量中有 `_omg_state` 数据（在控制台执行 `getVariables({type:'chat'})._omg_state`）
- **样式不生效**：检查酒馆页面 `<head>` 中是否有 `script_id` 属性的 `<div>` 包含脚本样式

## Phase 4: 样式工坊

### 前置条件

1. 脚本已加载成功，且可打开“状态栏管理器”
2. 已有至少 1 条定义（用于预览占位符）

### 可复现测试步骤

1. 打开管理器 → 切换到“样式工坊”
2. 确认左侧显示“内置样式”和“自定义样式”两组列表
3. 点击“新建样式”，输入名称并保存
4. 编辑 HTML/CSS，观察右侧预览区是否实时更新
5. 在模板中使用 `{{label}}` / `{{value}}`，确认占位符可见且被正确替换
6. 从内置样式执行“复制”，确认生成可编辑副本
7. 点击“保存”，关闭并重新打开管理器，确认样式仍在（持久化）
8. 导出样式 JSON，随后导入同一文件，确认列表和内容一致
9. 输入危险 CSS（如 `*{display:none}`），确认出现安全警告

### 期望结果

- 编辑器、预览、保存、导入导出链路可用
- 风险 CSS 有明确提示，不会静默失败
- 重开管理器后数据不丢失

## Phase 5: 布局编排器

### 前置条件

1. 已在数据工作室创建至少 2 个条目（用于放置到布局）
2. 管理器可正常打开

### 可复现测试步骤

1. 打开管理器 → “布局编排器”
2. 新建布局，确认列表出现新项并可选中
3. 添加容器节点与条目节点
4. 对节点执行上移/下移/删除
5. 切换布局模式（flex-row / flex-col / grid / absolute / custom）
6. 修改 gap、padding、alignment 等属性，观察预览实时变化
7. 切换到 JSON 编辑模式，修改并保存，再切回可视化模式确认回显
8. 导出布局 JSON 后导入，确认结构一致

### 期望结果

- 布局树、属性面板、预览保持一致
- JSON 模式与可视化模式双向可用
- 导入导出不会破坏节点层级

## Phase 6: 数据中心

### 前置条件

1. 聊天变量里存在 `_omg_state`（可用 Phase 3 的注入脚本）
2. 已创建至少一个角色分类和共享分类定义

### 可复现测试步骤

1. 打开管理器 → “数据中心”
2. 在左侧切换“共享数据/角色数据”
3. 在角色列表确认可见 `char_id`
4. 点击眼睛图标切换 `isPresent`
5. 新增角色并编辑字段；删除非 `char_user` 角色
6. 编辑数字/布尔/文本/JSON 类型条目
7. 新增条目时检查推荐输入是否给出已定义键
8. 保存后关闭并重开管理器，确认数据仍保留
9. 验证未知 key 是否进入“其他”分组

### 期望结果

- 编辑路径完整可用（新增、修改、删除、保存）
- 类型编辑行为与定义约束一致
- 存储回写稳定，重开后数据一致

## Phase 7: 管理器整合

### 前置条件

1. 五大模块均能在侧边栏看到
2. 已有至少一条可用于深度链接的定义/样式数据

### 可复现测试步骤

1. 依次切换五大模块标签，确认内容区正确刷新
2. 在 DataStudio 使用“渲染样式”跳转，确认定位到样式工坊
3. 在各模块点击 `?` 帮助按钮，确认弹出说明
4. 关闭帮助浮窗后继续操作主界面
5. 使用脚本按钮“打开管理器”与扩展菜单按钮分别打开管理器，确认都可用

### 期望结果

- 导航切换稳定，无空白页
- 深度链接与帮助系统可用
- 多入口打开管理器行为一致

## Phase 8: 系统配置

### 前置条件

1. 定义、样式、布局、配置至少各有一条数据
2. 可访问系统配置模块

### 可复现测试步骤

1. 打开“系统配置”并切换“当前状态/系统配置”视图
2. 开启 JSON 编辑：输入合法 JSON 并保存，随后输入非法 JSON 验证报错
3. 主题组合执行新建/编辑/删除
4. 叙事快照模板执行 CRUD，并切换启用状态
5. 在“备份与迁移”选择多模块导出
6. 使用导出文件执行导入，确认对应模块恢复
7. 执行“恢复出厂”并完成双重确认
8. 打开内置使用指南并检查内容加载

### 期望结果

- 系统配置的读写与校验行为明确、可预期
- 备份/导入/恢复出厂可闭环
- 使用指南可被非开发用户直接阅读
