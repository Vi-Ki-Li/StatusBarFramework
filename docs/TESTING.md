## Phase 8.10：角色切换顺滑性（新增）

目标：验证角色切换不再触发整栏闪烁式刷新，切换行为可通过运行时日志稳定观测。

### 前置条件

1. 场景已切到目标聊天与目标主题组合。
2. 状态栏存在至少 2 个角色 tabs。
3. 控制台可读取 `[通用状态栏框架][toggle]` 日志。

### 可复现步骤

1. 记录切换前基线：`tabCount`、状态栏根节点数量、DOM 节点总量。
2. 点击非当前激活角色 tab 一次。
3. 检查运行时日志是否按顺序出现 `switch-start`、`switch-commit`、`switch-end`。
4. 校验切换后激活 tab 已变更为目标角色，且状态栏根节点仍为同一节点（不应整栏 replace）。
5. 校验切换后 `tabCount` 与 DOM 节点总量无异常突增。
6. 连续切换 5 次（A-B-A-B...），确认每次均有完整日志且无明显闪烁。

### 通过标准

1. 单次切换日志完整且有序（start -> commit -> end）。
2. 默认走 in-place 提交（仅在异常时允许 fallback）。
3. 无肉眼可见整栏闪烁或明显重排卡顿。

### 失败时记录

1. 失败步骤编号。
2. 对应日志片段（至少包含一次完整切换前后）。
3. 根因分类：场景/数据/事件/样式/渲染。

### 测试执行约束（避免无效结论）

- 以截图可见结果为准，不用“代码存在”替代“用户可见”。
- 允许代码辅助理解，但必须以截图事实为先：截图不可见 = 用户不可见。
- 每次只做一个动作（输入或点击）后立即截图，必须有 before/after。
- 模块截图命名必须固定：`<序号>-<模块>-before/after-<desktop/mobile>.png`。
- 若图像重复或覆盖，视为测试失败，需重测该模块。
- 截图与测试 JSON 仅允许存放在 `session-state`，不得写入仓库目录（如 `docs/artifacts/`）。

### 截图降噪规则（新增）

- `after` 必拍：每个点击动作后都要截图。
- `before` 分级：
  - 必拍：首次进入模块、关键状态切换前、阻塞留证前。
  - 可复用：若“上一张 after 与当前点击前状态一致”，可将上一张 after 作为本步 before 引用，不重复出图。

## 通用调试技巧

- **找不到 extensionsMenu 按钮**：检查 `#extensionsMenu` 元素是否存在，可能在不同酒馆版本中 ID 不同
- **管理器打不开**：查看控制台是否有 JS 错误
- **状态栏不显示**：确认聊天变量中有 `_omg_state` 数据（在控制台执行 `getVariables({type:'chat'})._omg_state`）
- **样式不生效**：检查酒馆页面 `<head>` 中是否有 `script_id` 属性的 `<div>` 包含脚本样式

## Phase 4: 样式工坊

### 前置条件

1. 脚本已加载成功，且可打开“状态栏管理器”
2. 已有至少 1 条定义（用于预览占位符）

### 可复现测试步骤

1. 打开管理器 → 切换到“样式工坊”
2. 确认左侧显示“内置样式”和“自定义样式”两组列表
3. 点击“新建样式”，输入名称并保存
4. 编辑 HTML/CSS，观察右侧预览区是否实时更新
5. 在模板中使用 `{{label}}` / `{{value}}`，确认占位符可见且被正确替换
6. 从内置样式执行“复制”，确认生成可编辑副本
7. 点击“保存”，关闭并重新打开管理器，确认样式仍在（持久化）
8. 导出样式 JSON，随后导入同一文件，确认列表和内容一致
9. 输入危险 CSS（如 `*{display:none}`），确认出现安全警告

### 期望结果

- 编辑器、预览、保存、导入导出链路可用
- 风险 CSS 有明确提示，不会静默失败
- 重开管理器后数据不丢失

### 代理执行建议

- 优先通过注入数据 + 批量点击完成，不手工逐字输入字段。
- 每步完成后记录单条断言（DOM 文本/按钮状态/错误提示是否出现）。
