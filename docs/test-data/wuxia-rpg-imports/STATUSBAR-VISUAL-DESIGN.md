# 状态栏视觉设计指南（酒馆场景）

## 为什么你会觉得“不好看”

从当前武侠示例看，主要问题通常不是“配色错了”，而是信息层级不稳定：

1. 标签和数值视觉权重接近，读者不知道先看哪里。
2. 同屏高对比组件太多，每块都在抢注意力。
3. 分组节奏不明显，世界信息/角色信息/战斗信息混在一起。
4. 主题只改颜色，不改组件语义，导致“换皮不换阅读路径”。

## 好看的核心：先可读，再风格

1. 先定义阅读顺序

- 第一眼：关键数值（气血、内力、时辰、风闻）
- 第二眼：状态变化（中毒、心法、身位）
- 第三眼：叙事氛围（材质、纹理、色调）

1. 再定义组件语义

- 大文本信息：牌匾/卷轴（地域、兵器、心法）
- 数值资源：进度槽（气血、内力、体力）
- 布尔状态：双态旗（中毒）
- 背景指标：尺状进度（风闻、轻功位）

1. 最后统一主题语言

- 颜色控制在主色 3 个 + 强调色 1 个
- 圆角控制在 2-3 档
- 阴影只用于层级，不用于装饰堆砌

## 实操清单（每次做主题都走一遍）

1. Data Studio

- 条目按语义分组：世界/角色/战斗
- 每个条目指定默认 `uiType`

1. Style Workshop

- 至少准备 6-8 个可复用样式单元
- 每个单元只承担一种阅读任务

1. Layout Composer

- 总览版：3 行结构（世界主行 -> 角色身份 -> 战斗读条）
- 紧凑版：徽章行 + 核心读条 + 状态行

1. Theme Combo

- 提供两套主题：总览（剧情）与紧凑（战斗）
- 保存后立即绑定并刷新确认

## 视觉验收标准

1. 一秒读值

- 关键数值字号明显大于标签（建议 >= 1.5x）

1. 三秒懂状态

- 用户能在 3 秒内说出：当前时空、角色健康、战斗风险

1. 五秒记风格

- 不看文字也能识别题材（武侠/科幻/校园）
