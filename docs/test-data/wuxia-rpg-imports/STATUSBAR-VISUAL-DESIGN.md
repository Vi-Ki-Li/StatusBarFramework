# 状态栏视觉设计指南（酒馆场景）

## 为什么你会觉得“不好看”

从当前武侠示例看，主要问题通常不是“配色错了”，而是信息层级不稳定：

1. 标签和数值视觉权重接近，读者不知道先看哪里。
2. 同屏高对比组件太多，每块都在抢注意力。
3. 分组节奏不明显，世界信息/角色信息/战斗信息混在一起。
4. 主题只改颜色，不改组件语义，导致“换皮不换阅读路径”。

## 本轮主题方向：竹、墨、酒、剑

1. 竹（结构）

- 视觉意图：秩序、节奏、筋骨。
- 手段：竖向边线、分段边框、竹节式重复纹理、偏青绿中低饱和。

1. 墨（基底）

- 视觉意图：沉静、留白、信息承托。
- 手段：墨灰渐变、深浅层级阴影、胶囊墨印标签。

1. 酒（情绪）

- 视觉意图：温度、呼吸感、人物性格。
- 手段：琥珀色高光、小范围浮动动效、圆润容器角。

1. 剑（决断）

- 视觉意图：方向性、锋利感、战斗节奏。
- 手段：斜切进度条、冷灰金属线、尖角/折线 SVG 分割。

## 视觉强化手段（不止 icon 和 border）

1. 内联 SVG

- 可直接写进 style template：
 	- 剑锋形进度蒙版
 	- 云雾曲线装饰
 	- 印章/剑纹符号

1. 纹理与材质

- repeating-linear-gradient 做竹简纹和卷轴纹。
- radial-gradient 做墨晕/灯下纸面。

1. 层级系统

- 同一页固定 2-3 档圆角、2 档阴影、1 套描边粗细。

1. 微动效

- 常驻动效建议仅 1-2 处（如酒气浮泡、焦点脉冲）。
- 时长建议 150-250ms（交互）与 1.2-2.0s（呼吸/漂浮）。

## 好看的核心：先可读，再风格

1. 先定义阅读顺序

- 第一眼：关键数值（气血、内力、时辰、风闻）
- 第二眼：状态变化（中毒、心法、身位）
- 第三眼：叙事氛围（材质、纹理、色调）

1. 再定义组件语义

- 大文本信息：牌匾/卷轴（地域、兵器、心法）
- 数值资源：进度槽（气血、内力、体力）
- 布尔状态：双态旗（中毒）
- 背景指标：尺状进度（风闻、轻功位）

1. 最后统一主题语言

- 颜色控制在主色 3 个 + 强调色 1 个
- 圆角控制在 2-3 档
- 阴影只用于层级，不用于装饰堆砌

## 本次修正重点：容器同形制策略

1. 同一容器只使用同一类样式单元

- 世界层统一为竹牌（`su_wx_bamboo_plaque`），保证信息起始层节奏一致。
- 角色数值层统一为剑槽（`su_wx_sword_bar`），读值路径稳定。
- 战斗状态层统一为卷轴行（`su_wx_scroll_line`），避免布尔态和文本态混搭失衡。
- 技能描述层统一为卷轴行，技能对象层单独占一行使用 `su_wx_json_chip`。

1. 先分层，再强调

- 先把容器视觉秩序排对，再在跨容器层级做风格强调。
- 强调组件（如酒葫芦、印章）不应和基础读值组件混放在同一容器。

## 关于 Single/List 的历史与后续

1. 历史决策

- 在 DEC-007 中，`entry_type`（Single/List）与 `parts` 被移除。
- 原因是当前更新链路改为 JSON Patch，解析器不再需要提前声明结构。

1. 现状限制

- 数据工作室现在是“显示/验证元数据”，而不是“对象结构建模器”。
- 复杂对象可通过扁平键 + JSON 摘要文本落地，但可编辑性和可视化粒度有限。

1. 建议演进方向

- 新增可选 `structure_hint`（不影响 Patch 执行）：`single`、`list`、`map`。
- 新增 `render_parts`（仅用于 UI 渲染拆分），支持对象字段映射到多个样式位。
- 保持向后兼容：不配置时维持当前行为。

## 实操清单（每次做主题都走一遍）

1. Data Studio

- 条目按语义分组：世界/角色/战斗
- 每个条目指定默认 `uiType`

1. Style Workshop

- 至少准备 6-8 个可复用样式单元
- 每个单元只承担一种阅读任务

1. Layout Composer

- 总览版：3 行结构（世界主行 -> 角色身份 -> 战斗读条）
- 紧凑版：徽章行 + 核心读条 + 状态行

1. Theme Combo

- 提供两套主题：总览（剧情）与紧凑（战斗）
- 保存后立即绑定并刷新确认

## 视觉验收标准

1. 一秒读值

- 关键数值字号明显大于标签（建议 >= 1.5x）

1. 三秒懂状态

- 用户能在 3 秒内说出：当前时空、角色健康、战斗风险

1. 五秒记风格

- 不看文字也能识别题材（武侠/科幻/校园）
