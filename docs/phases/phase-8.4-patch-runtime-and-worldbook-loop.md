# Phase 8.4 补丁运行时与世界书闭环

## 需求

- AI 输出补丁应被脚本自动解析并写入状态，而不是仅显示在对话文本中。
- 补丁输出应可自动隐藏（正则由框架管理），并可在系统配置中开关。
- 角色路径更新应自动出现在数据中心（无需手动先创建角色）。
- 样式工坊自定义样式与全局主题应真实作用于状态栏渲染。
- 删除楼层后，原始数据查看器应尽量回溯到最近快照。
- 叙事模板应按每轮变化注入到世界书（风格常驻 + 变化轮播）。

## 讨论与边界

- 本轮仅支持标签协议：`<OmgPatch>...</OmgPatch>`。
- 不兼容 Markdown 代码块补丁格式（按需求明确关闭 fallback）。
- 楼层删除回溯采用“最近可用快照”策略，不实现严格逐楼重放。

## 研究

- 现有代码中 `extractPatchFromMessage`、`mergePatches` 已具备基础能力，但未接入消息事件链路。
- 渲染器未加载自定义样式单元与全局主题，导致样式看似“未应用”。
- `/characters/<char_id>/...` 写入仅创建 `characters`，未同步 `_characters` 列表，数据中心不可见。
- 叙事模板仅有 CRUD，未接入世界书注入运行时。

## 规划

1. 建立 patch runtime：消息监听、提取、合并、保存、重渲染。
2. 增加补丁隐藏正则自动管理与系统配置开关。
3. 修复角色路径自动创建角色元信息。
4. 修复样式渲染链（自定义样式 + 全局主题注入）。
5. 新增楼层删除快照回溯。
6. 叙事模板注入世界书（风格条目常驻、变化条目每轮覆盖）。

## 执行记录

- `src/通用状态栏框架脚本/data/json-patch.ts`
  - 改为仅解析 `<OmgPatch>...</OmgPatch>`。
  - 新增 `PATCH_TAG_OPEN/PATCH_TAG_CLOSE` 常量。

- `src/通用状态栏框架脚本/data/worldbook-inject.ts`
  - 世界书总览提示改为强制标签协议，明确禁用 Markdown 代码块补丁。

- `src/通用状态栏框架脚本/data/patch-runtime.ts`
  - 新增运行时模块：监听 AI 消息提取补丁、合并状态、保存快照并刷新状态栏。
  - 新增自动隐藏补丁正则管理（`OMG_StatusBar_HidePatch`）。
  - 新增 `setPatchHideRegexEnabled` 供系统配置实时开关。
  - 新增楼层删除后的最近快照回溯逻辑。
  - 新增叙事模板世界书注入：
    - `OMG_Narrative_StylePrompt`（常驻风格）
    - `OMG_Narrative_Changes`（本轮变化）

- `src/通用状态栏框架脚本/data/variables.ts`
  - 系统配置新增 `patchHideRegexEnabled`（默认 `true`）。

- `src/通用状态栏框架脚本/data/merge.ts`
  - 角色路径更新时自动补 `_characters` 元信息（含 `char_user` 默认名“用户”）。

- `src/通用状态栏框架脚本/renderer/status-bar.ts`
  - 渲染器加载自定义样式单元和全局主题。
  - 状态栏根节点兼容 `omg-statusbar` 类名。

- `src/通用状态栏框架脚本/index.ts`
  - 接入 patch runtime 生命周期。
  - 新增脚本按钮：`刷新状态栏`。

- `src/通用状态栏框架脚本/manager/modules/SystemConfig.vue`
  - 世界书条目管理区新增“自动隐藏补丁标签”开关，并实时同步正则启停。

## 验证

- 类型检查：改动文件无编译错误。
- 行为验证待人工回归：
  1. 注入世界书后，AI 以 `<OmgPatch>` 输出补丁。
  2. 补丁文本在显示层被正则隐藏。
  3. 状态栏和数据中心同步更新（共享 + 角色路径）。
  4. 删除楼层后触发快照回溯提示。
  5. 开关关闭时，补丁文本不再自动隐藏。
  6. 叙事风格与本轮变化条目在框架世界书中可见。

## 风险与后续

- 叙事注入当前取“第一条模板”为活动模板，后续建议增加显式“活动模板”选择。
- 快照回溯为最近快照恢复，复杂删楼场景下可能与用户预期存在差异。
- 可考虑为补丁提取失败增加可视化诊断面板（便于用户定位格式错误）。
