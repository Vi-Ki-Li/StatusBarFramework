# Phase 8.13：新增需求讨论与研究启动

日期：2026-03-26

## 1) 需求（Requirement）

本阶段处理 2026-03-26 新增需求的“讨论→研究→规划”起步，不直接进入全面实现。  
原始需求已归档于 `docs/REQUIREMENTS-RAW.md`（新增补充段落）。

## 2) 讨论（Discussion）

### 优先级分层

P0（先做）：

1. 对象/数组渲染能力（尤其对象优先的可维护表达）。
2. 第一层数据模型重构（从“共享/角色”硬分类转向“模板化对象层”理念）。

P1（紧随其后）：

1. 状态栏悬浮开关（非固定聊天内）。
2. 管理器记忆上次页面 + 可拖拽缩放。
3. 状态栏本体快速切换主题/布局按钮。

P2（在 P0/P1 稳定后）：

1. 布局级样式覆盖（同样式单元多配色复用）。
2. 布局级字体调节能力。
3. 系统配置-世界书条目内联查看/编辑增强。

### 边界

1. 不在未完成模型设计前直接写对象/数组渲染实现，避免接口反复推翻。
2. 先形成数据契约，再做 UI 能力扩展，防止“先 UI 后模型”导致的返工。

## 3) 研究（Research）

### 已知现状

1. 当前渲染主链条目粒度偏“标量/扁平 key”，复杂对象多依赖手工编排。
2. `LayoutNode` 已支持容器级 `customCss/width/height/styleOverride`，具备布局级覆盖的落点。
3. 管理器已有模块化路由，可扩展“记忆上次页面”和窗口状态持久化。
4. 系统配置已有世界书子模块基础，可增“条目内容内联编辑”。

### 关键风险

1. 数据第一层模型变更会影响：数据中心、定义工作室、渲染器、主题组合、导入导出兼容。
2. 对象/数组渲染若直接引入深层递归，可能导致布局编辑器和样式工坊复杂度骤升。

## 4) 规划（Planning）

- [ ] 定义“对象优先”渲染契约（对象、数组、标量三类最小统一接口）
- [ ] 设计第一层数据抽象（模板化对象层）并拟定兼容迁移策略
- [ ] 评估布局级样式覆盖/字体控制的最小可交付 API（节点级覆盖优先）
- [ ] 设计状态栏悬浮模式与管理器窗口状态持久化方案
- [ ] 设计系统配置-世界书条目内联编辑交互与保存模型
- [ ] 输出跨模块影响清单和分批实施顺序（P0→P1→P2）

## 5) 下一步建议

1. 先完成 P0 的数据契约与迁移策略文档（不改代码）。
2. 再按契约分两批实现：
   - 批次 A：对象/数组渲染 + 第一层模型；
   - 批次 B：悬浮模式、管理器窗口状态、快速切换按钮与世界书内联编辑。
