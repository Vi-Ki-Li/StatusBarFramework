# Phase 8.9 - Runtime Collapse And Tabs Rootcause Handoff

## 背景

本阶段目标是收敛三个运行时问题：

1. 总布局折叠（"布局：参考图复刻-全量"）应有展开和收起动画。
2. 分组折叠（layout group）应有展开和收起动画。
3. 多角色应显示角色切换栏（tabs）。

用户明确要求：不要继续堆兜底，要找根因。

## 用户提供的关键证据

用户给出的状态原始数据（聊天变量 `_omg_state`）显示：

1. `_characters` 中有两个角色：`char_kaito`、`char_yuki`。
2. 两者 `isPresent` 都是 `true`。
3. `characters` 中两个角色均有大量字段值。
4. `shared` 也有值（`面板.时间`、`面板.地点`、`面板.天气`）。

这意味着：按现有渲染逻辑，tabs 理论上必须出现；若仍不出现，问题不在“有没有 shared 数据”。

## 已实施改动（但用户回报仍失败）

文件：`src/通用状态栏框架脚本/renderer/status-bar.ts`

1. tabs 候选逻辑增强：

- 从“仅 isPresent”扩展为“在场优先，不足两角色则回退全部角色”。

1. hasData 判定增强：

- 增加 `state.characters`，避免仅角色数据时被判空。

1. details 收起动画接管：

- 在 summary 点击时，若处于 open 状态，阻止默认 close。
- 先执行 body 的 `max-height/opacity` 收起过渡，再将 `details.open=false`。

1. 调试日志增强：

- 注入日志增加 `tabCount`。

用户最新反馈：

- 总布局收起仍无动画。
- 分组头仍无动画。
- 角色栏仍不可见。

结论：以上改动未在用户实际场景里达成目标，需新会话按运行时证据继续深挖。

## 当前高概率根因假设（按优先级）

1. 交互目标元素与动画 body 的命中不稳定

- summary 点击时，`querySelector('.omg-sb__section-body, .omg-sb__layout-group-body')` 可能命中非预期节点或高度为 0。
- 导致“逻辑执行了，但视觉无变化”。

1. 事件绑定生命周期与重渲染竞争

- 每次渲染都会 `off('.omg-sb')` + 重新绑定。
- 可能在某些消息事件链中出现“绑定后立即替换 DOM”，导致本次点击不在同一批节点上。

1. tabs 不是“未生成”，而是“生成后被遮挡/不可见”

- 需用运行时 DOM 直接确认 `.omg-sb__tab` 是否存在。
- 若存在，则是样式/层叠/插入位置问题；若不存在，才是数据到渲染链路问题。

1. 注入位置与容器替换导致观察错位

- 当前注入在 `#chat .mes.last_mes` 后，可能不是用户正在观察的那条楼层。
- 需要确认页面是否同时存在多个状态栏实例被先后移除/重建。

## 历史兼容问题说明（必须保留）

`面板.时间` 这类扁平键并非本次用户新引入，而是历史生成策略导致。

当前 patch path 写法（示例）：

- `/shared/面板.时间`

在 JSON Patch 转 lodash 路径时，会保留为单段键名 `面板.时间`，不是嵌套对象路径。

如果希望写嵌套对象，必须使用：

- `/shared/面板/时间`
- `/characters/char_xxx/生存/精力值`

该问题会影响定义路径命中与展示一致性，但不是本次“tabs 不显示”的充分解释。

## 开发约束回顾

1. 结构问题在 layout/runtime 解决，不把结构规则硬塞到 style/globalTheme。
2. 折叠先保证行为一致，再谈动效质感。
3. 不再用“可能是共享数据”作为主假设，先以运行时 DOM 与日志证据为准。

## 交接结论

当前代码并未在用户场景中达成三项目标。下一会话应直接从“运行时证据采样”开始，而不是继续做抽象级修补。

### 观察项（未改码）：角色切换体感卡顿/短暂“刷新态”

现象描述：切换角色时，状态栏会短时间出现接近整栏刷新时的视觉状态，体感表现为不自然、发卡。

初步判断（按优先级）：

1. 切角色触发了“整栏重建”而非“局部更新”，导致 DOM 大量替换与样式重算。
2. 重建期间存在过渡样式基线重置（例如 opacity/max-height/active class 的瞬时回退）。
3. 事件重新绑定与动画起止时序重叠，造成一次切换内出现两段视觉状态。

建议改进方向（先方案，暂不改代码）：

1. 渲染策略分层：
	- 角色切换优先走“active tab + 内容区局部切换”，避免整栏 `renderStatusBar` 全量重建。
2. 过渡策略稳定化：
	- 将切换动画限定在内容容器，保持外层结构与尺寸稳定；
	- 在切换期间短暂冻结可能被重置的折叠容器样式，防止闪回到初始态。
3. 事件策略去重：
	- 保证一次切换只触发一次有效渲染提交（可通过帧内合并或微任务去重）。
4. 观测与验收：
	- 新增角色切换专用日志字段：`switch-start`、`switch-commit`、`switch-end`、`rerender-count`；
	- 目标是单次切换 `rerender-count=1`，且不出现整栏闪烁。

建议落地位置：后续可在 phase-8.10（性能与交互顺滑）单独立项，避免与本阶段根因修复混杂。
